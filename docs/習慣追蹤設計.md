# 習慣追蹤功能設計文件

## 概覽

習慣追蹤（Habit Tracker）是 DailyFlow 三大核心模組之一，讓使用者能夠建立每日習慣清單、打卡記錄、追蹤連續達成天數，並透過視覺化圖表了解長期執行狀況。

---

## 資料模型

### Habit 物件結構

```typescript
interface Habit {
  id: number;          // Unix timestamp（建立時間，作為唯一識別碼）
  name: string;        // 習慣名稱，最多 20 字元
  icon: string;        // Emoji 圖標（⭐ / 🌅 / 💧 / 📖 / 🏃 / 🧘 / 🍎 / 💊）
  color: string;       // 背景色 HEX，從預設 5 色循環取用
  streak: number;      // 連續達成天數（動態計算）
  total: number;       // 累計打卡總天數
  checks: {            // 每日打卡記錄
    [dateKey: string]: 0 | 1;  // key 格式：YYYY-M-D
  };
}
```

### dateKey 格式

日期 key 採用非零填充格式，例如 `"2026-6-11"`，與 JavaScript `Date` 物件的月/日直接對應，避免格式轉換。

### 儲存位置

| 層級 | 機制 | 說明 |
|------|------|------|
| 本地 | `localStorage["df_habits"]` | JSON 序列化，離線時主要資料來源 |
| 雲端 | Supabase `habits` 表 | 跨裝置同步 |

## 核心邏輯

### Streak 計算演算法

連續達成天數（streak）採用**動態回溯計算**，每次打卡後重新計算，不採用增量式 +1/-1：

```javascript
function calcStreak(checks) {
  let streak = 0;
  const d = new Date(today);

  // 若今天尚未打卡，從昨天開始回溯
  if (!checks[dateKey(d)]) {
    d.setDate(d.getDate() - 1);
  }

  // 向過去逐日確認是否有打卡
  while (checks[dateKey(d)]) {
    streak++;
    d.setDate(d.getDate() - 1);
  }

  return streak;
}
```

**設計決策**：動態計算確保以下情況正確處理：
- 補打歷史日期時，streak 能正確反映連續狀態
- 跨天未打卡後，streak 歸零而非保留舊值
- 取消打卡後，streak 立即重算

### 總計數（total）計算

`total` 為累計打卡總天數，採用增量方式：
- 打卡時：`total++`
- 取消打卡時：`total = Math.max(0, total - 1)`

## UI 元件設計

### 1. 今日統計橫幅（habit-stat）

位於習慣週視圖上方，顯示當日完成比例。

```
┌─────────────────────────────────────┐
│  2/3                           💪   │
│  今日習慣已完成                      │
└─────────────────────────────────────┘
```

| 完成狀態 | Emoji |
|---------|-------|
| 0 個完成 | 🎯    |
| 部分完成 | 💪    |
| 全部完成 | 🎉    |

### 2. 本週打卡視圖（week-checks）

習慣卡片中央，顯示當週 7 天（週日～週六）的打卡圓點：

- 已打卡：藍色底 + ✓ 符號（`.wc.done`）
- 今日邊框：藍色加粗邊框（`.wc.today-marker`）
- 可點擊補打或取消

### 3. 過去 4 週熱力圖（heat-wrap）

可折疊展開，位於每個習慣卡片底部。

```
過去 4 週記錄 ↓

[  ][  ][  ][  ][✓][  ][  ]   ← 4週前
[  ][✓][✓][  ][✓][✓][  ]   ← 3週前
[✓][✓][  ][✓][✓][✓][✓]   ← 2週前
[✓][✓][✓][✓][✓][  ][今]   ← 本週

4 週前                    本週
```

- 格子顏色：未打卡（邊框灰）、已打卡（半透明藍）、今天（深藍）
- 排列：7 欄（週日～週六）× 4 列（最舊至最新）
- 起始點：4 週前的週日（與週視圖對齊）

## 雲端同步策略

### 冷啟動同步

應用啟動時呼叫 `syncFromCloud()`，從 Supabase 拉取最新習慣資料，覆蓋本地 `localStorage`。

### 操作即時同步

每次打卡（`toggleHabitDay`）、新增/編輯/刪除習慣後，立即 `await` 更新 Supabase，失敗則靜默（不影響本地狀態）。

### 離線降級

Supabase 請求失敗時 toast 顯示「離線模式，資料存本地」，所有操作仍寫入 `localStorage`，待下次連線時可手動按同步鈕補傳。

## 已知限制與未來方向

| 項目 | 現況 | 可能改進方向 |
|------|------|-------------|
| 歷史視窗 | 僅顯示過去 28 天熱力圖 | 支援月份切換或全年視圖 |
| 週期設定 | 固定每日習慣 | 支援「每週 N 次」等彈性目標 |
| 統計報告 | 基本完成率 | 月度/年度統計圖表 |
| 提醒通知 | 統一推播通知 | 每個習慣可設定獨立提醒時間 |
| 資料匯出 | 無 | CSV / JSON 匯出 |
